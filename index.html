Why This Exists

The Problem

The frontier AI landscape shifts faster than static keyword lists can capture.

The landscape is deep, broad, and arcane. A single role might require fluency across reinforcement learning theory, distributed training infrastructure, evaluation methodology, and annotation pipeline operations — each with its own vocabulary, tools, and research lineage. What counted as cutting-edge 18 months ago is now table stakes. What's emerging today won't appear in job descriptions for another year.

This creates three compounding issues:

Terminology fragmentation

The same work gets called "RLHF," "preference optimization," "alignment training," or "post-training" depending on whether someone came from a research lab, AI startup, or big tech ML team. Standard searches pick one term and miss the others.

Search saturation

Obvious keywords surface the same candidates to every recruiter. Differentiation requires depth that's time-prohibitive to build manually for each role.

Workflow friction

Even with good keyword lists, composing effective searches requires understanding how terms relate. Manual approaches put this cognitive load on sourcers, slowing adoption and creating inconsistent results.

What You Get

The template generates a standalone HTML file with three components:

Role Summary

1 per role

Quick reference cards showing core function, technical domain, key deliverables, and experience level extracted from the job description.

Archetypes

3-4 per role

Heuristic anchor points — practical starting profiles that orient your search before you dive into the full library. Each represents a distinct professional path to the role (lab post-training lead vs. ML evaluation infrastructure builder), names WHO you're looking for, and provides a recipe (which boolean clusters to combine) with a WHY (how the clusters relate to one another and narrow the search).

Search Library

4-6 blocks

The complete term library, pre-expanded to catch candidates regardless of which vocabulary they use. Organized by competency area (blocks), search angle (Concepts, Methods, Tools), and precision tier (Broad, Established, Recent, Specific). Recent and Specific clusters contain terminology generic searches miss — that's where differentiation comes from.

Blocks 4-6 per role

Core competency areas. A viable candidate has signal in all blocks — the variation is in depth, not presence.

Sub-blocks 3 per block

Search angles that capture different ways expertise appears on profiles:

Sub-block What it captures Where it appears
Concepts Field vocabulary Team/org descriptions
Methods Techniques, processes Skills sections
Tools Libraries, frameworks, platforms GitHub, docs references
Clusters 4 per sub-block

Precision tiers that control recall vs. specificity:

Cluster Purpose
Broad High recall — catches anyone in the space
Established Mainstream terms with 2+ years of usage
Recent Cutting-edge terminology (last 12-18 months)
Specific Implementation-level terms signaling hands-on experience

How to Use

1

Download

Get the template file (.md) from the link above.

2

Generate

Open Claude (Opus 4.5 recommended). Paste the template and your job description. Save the generated HTML file.

3

Use

Open the HTML in your browser. Follow archetype recipes to identify which clusters to combine. Copy clusters from the Search Library. Search.

Recommended Models

Use Case Model Notes
Full template execution Claude Opus 4.5 Required for reliable HTML output generation
Fast iteration Claude Sonnet 4.5 May work for simpler roles; verify HTML output
6

Output Checklist

Before using generated output, verify structure:

  • 4-6 blocks, each a core competency area
  • 3 sub-blocks per block (Concepts, Methods, Tools)
  • 4 clusters per sub-block (Broad, Established, Recent, Specific)
  • 8-20 terms per cluster
  • 3-4 archetypes with recipes and WHY explanations
